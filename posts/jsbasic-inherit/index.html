Inheritance

The purpose of prototypes is to form the prototype chain:

  • Reference types are all objects, and each object has a prototype.
  • Prototypes are also objects, and they also have their own prototypes, forming a prototype chain layer by layer.

The purpose of the prototype chain is inheritance:

  • When accessing an object's properties, if the property cannot be found in the object itself, the search is performed up the prototype chain layer by layer.
  • In other words, one object can access the properties of other objects.

The purpose of inheritance is property sharing:

  • The first benefit is code reuse (literally).
  • The second benefit is extensibility. Different objects can inherit the same properties, or they can define their own properties.

ES5 Inheritance Implementation

Prototype Chain Inheritance

Principle:

  • Use the parent class instance as the prototype of the child class

Here, both son1 and son2 share the same reference to the parent’s names array.

Advantages

  • Parent methods can be reused.
  • Both parent properties and methods are accessible to child instances.

Disadvantages

  • Reference-type properties from the parent are shared by all child instances, which can cause data tampering between instances.
  • Child instances may inherit unnecessary parent properties, leading to inefficiency.
  • Since the child’s inheritance is achieved by assigning Son.prototype = new Parent(), there’s no way to pass arguments to the parent constructor during child instantiation. → Parent constructor properties cannot be initialized with custom values per child instance.

Constructor Inheritance (Borrowing Constructor)

Principle:

  • Use the parent’s constructor function to enhance the child instance.
  • By leveraging call or apply to change the value of this, the parent’s constructor logic is executed in the child’s context.
  • Since the parent’s properties are bound to this, the child instance naturally inherits them.

This is the only inheritance pattern that does not involve prototype.

Here, unlike prototype chain inheritance, each child instance gets its own color array, so modifications do not affect other instances.

Advantages

  • No shared reference issue: Each child has its own copy of the parent’s reference-type properties, avoiding unintended modifications across instances.
  • Parameter passing: Child constructors can pass arguments to the parent constructor, enabling flexible property initialization.

Disadvantages

  • No prototype inheritance: Child instances cannot inherit methods/properties defined on the parent prototype. If such methods need to be inherited, they must be explicitly defined inside the constructor, which defeats the purpose of prototypes.
  • No method reuse: Methods are copied into each child instance separately, violating the principle of code reuse.
  • Performance overhead: Every child instance creates its own copy of the parent’s methods, leading to potential memory inefficiency.

Combination Inheritance

A combination of prototype chain inheritance and constructor borrowing inheritance.

Principle:

  • Use the prototype chain to inherit properties and methods from the parent prototype.
  • Use constructor borrowing to inherit instance properties.

Advantages

  • Parent methods are reusable.
  • Parent reference-type properties are not shared across instances.
  • Child constructors can pass arguments to the parent constructor.
  • Supports inheritance of both instance properties and prototype methods.

Disadvantage

  • The parent constructor is called twice:
    • Once when setting up the child prototype (SubType.prototype = new SuperType())
    • Once when calling SuperType.call(this, ...) inside the child constructor
    • This results in duplicate properties/methods being created unnecessarily, leading to performance inefficiency.

Prototypal Inheritance

In ES5, Object.create() provides a standardized way to implement prototypal inheritance.

  • Before ES5, this pattern could be simulated using a helper function that creates an empty intermediary object whose prototype is set to the input object.

Drawbacks

  • Shared reference-type properties: Reference values (e.g., arrays, objects) on the parent are shared across all child instances. Modifying one affects all others, similar to the issue in prototype chain inheritance.
  • No parameter passing: Cannot pass arguments to the parent during child instantiation, so parent properties cannot be initialized with custom values.

✅ ES5 Standardization

  • The ES5 method Object.create() replaces the custom object() function and provides a standardized, built-in way to achieve prototypal inheritance.

Parasitic Inheritance

Built on top of prototypal inheritance, this pattern defines a function specifically for wrapping the inheritance process.

Inside the function, a new object is created based on an existing one, and then enhanced in some way before being returned.

The resulting object not only inherits the parent’s properties and methods but can also be extended with additional functionality.

Drawbacks

The disadvantages are the same as prototypal inheritance:

  • Reference-type properties on the parent are shared by all instances.
  • There’s no way to pass parameters to the parent during instantiation.

Parasitic Combination Inheritance

Parasitic combination inheritance is a combination of parasitic inheritance and constructor borrowing inheritance.

It improves upon combination inheritance by calling the parent constructor only once, thereby avoiding the redundancy of calling it twice as in traditional combination inheritance.

Praise from JavaScript: The Definitive Guide

  • Highly efficient because it only calls the parent constructor once, avoiding the creation of unnecessary properties on Parent.prototype.
  • The prototype chain remains intact, so instanceof and isPrototypeOf work as expected.
  • Widely regarded by developers as the most ideal inheritance pattern for reference types, and it is the method used by many libraries.

ES6 inheritance is similar to parasitic composition inheritance. Essentially, ES6 inheritance is syntactic sugar for ES5 inheritance.

  • Differences between ES5 inheritance and ES6 inheritance.

    In ES5 inheritance essentially creates an instance of the child class first, then adds the parent class's methods to this.

    Inheritance in ES6 is different.

    • In an ES6 class, you first create an instance of the parent class's this object, then modify this in the child class's constructor.
    • Subclasses must call super in their constructor method; otherwise, an error will occur when creating a new instance. This is because subclasses don't have their own this object; instead, they inherit the parent class's this object and then modify it.

What's the difference between "classes" in ES5 and classes in ES6?

In ES5, classes are primarily defined using the constructor and prototype methods. In ES6, classes can be defined using the class constructor.

  • Classes must be called with new and cannot be executed directly.

    Executing a class will result in an error. However, classes in ES5 are essentially the same as ordinary functions, so execution will definitely work.

  • class does not hoist variables.

    The above example shows that the class method does not hoist the class definition to the top.

  • class cannot traverse the properties and methods on their instance prototype chains.

  • new.target property

    ES6 introduces a new.target property for the new command, which returns the constructor function the new command is applied to. If not called via new or Reflect.construct(), new.target returns undefined.

  • class class has static static method

    static methods can only be called from the class and will not appear on instances; if a static method contains the this keyword, this refers to the class, not the instance

    Static properties and methods declared with static are inherited by subclasses

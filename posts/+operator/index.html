What are the results of JS's {} + {} and {} + []?

Created2021-05-16|Updated2025-10-02|FrontendJS
|Post Views:

Preface

In JavaScript, the + symbol is very common and can be used in the following ways:

  • Addition of numbers (binary operator)
  • String concatenation (binary operator, with the highest precedence)
  • Unary plus, which can also be extended to mean forcing the operand to be converted into a numeric type

Another common syntax element is the curly braces {}, which also have two frequent uses:

  • Object literal definition
  • Block statements

Plus Operator +

In addition to the common cases mentioned above, the ECMAScript standard defines the following type conversion rules, which must be applied in order: operand + operand = result

  • Apply the ToPrimitive operation to convert both the left and right operands into primitive values.
  • After step 1, if either operand is of type String, the other operand is coerced into a string, and the operation is performed as string concatenation.
  • In all other cases, both operands are converted to the primitive type Number, and the operation is performed as numeric addition.

ToPrimitive Internal Operation

Since the plus operator can only be applied to primitive values, how are object values converted into primitive types?

When a JavaScript object is converted into a primitive value, the language uses the ToPrimitive algorithm.

This is an internal abstract operation defined by the specification, which JavaScript engines follow during execution.

In ECMAScript 6th Edition, §7.1.1, there is an abstract operation called ToPrimitive, which is used when an object needs to be converted into a primitive type.

This operation is not only used in the context of the + operator, but also for relational comparisons and equality checks.

The abstract syntax is: ToPrimitive(input, PreferredType?)

  • input represents the value to be converted.
  • PreferredType may be either Number or String, indicating the “preferred” type for the conversion.
  • The steps of the conversion process differ depending on PreferredType preference.

If no PreferredType is provided (the default case), then the hint value is set to default.

  • This hint acts as guidance for which primitive type should be targeted during conversion.
  • In most cases, JavaScript automatically applies this default hint during internal operations.

Conversion Algorithm

When an object is converted to a primitive value, the following logic is applied to determine which method on the object is called:

  • If obj[Symbol.toPrimitive] exists, then obj[Symbol.toPrimitive](hint) is invoked first.

  • Otherwise, the following rules apply:

    • When PreferredType is Number:

      Let input be the value to convert.

      • If input is already a primitive, return it directly.
      • Otherwise, if input is an object, call its valueOf() method. If this returns a primitive value, return it.
      • Otherwise, if input is an object, call its toString() method. If this returns a primitive value, return it.
      • Otherwise, throw a TypeError.
    • When PreferredType is String:

      Let input be the value to convert.

      • If input is already a primitive, return it directly.
      • Otherwise, if input is an object, call its toString() method. If this returns a primitive value, return it.
      • Otherwise, if input is an object, call its valueOf() method. If this returns a primitive value, return it.
      • Otherwise, throw a TypeError.
    • When PreferredType is not provided (i.e., the hint is default): The steps are the same as when PreferredType is Number.

Symbol.toPrimitive

Symbol.toPrimitive is a built-in Symbol value that serves as a function-valued property of an object. When an object is converted to a corresponding primitive value, this function is invoked.

When an object needs to be converted into a primitive (such as a number, string, or default type), the JavaScript engine calls this function and passes in a string parameter hint, which indicates the expected type of the resulting primitive value.

The hint parameter can take one of three values: "number", "string", or "default".

Symbol.toPrimitive has the highest priority in type conversion.

valueOf and toString Methods

In the design of JavaScript’s Object prototype, every object always has the two methods valueOf and toString.

  • These methods exist on all objects, although the order in which they are invoked during type conversion may differ.

Why does '' + test call valueOf instead of toString?

Rules of the + operator

  • When an object participates in the + operation, JavaScript internally invokes ToPrimitive(obj, hint="default") to convert the object into a primitive value.
  • If hint = "number" → it will first try valueOf, then toString.
    If hint = "string" → it will first try toString, then valueOf.
  • If hint = "default" → for most objects, this is treated the same as "number" (i.e., try valueOf first, then toString).

The only exception is the Date object, which prefers toString when hint = "default".

The case of '' + test

  • When JavaScript encounters '' + test, it needs to convert test into a primitive.
  • The operator is +, but it does not enforce a string-preferred conversion → so the hint is "default".
  • For ordinary objects, "default" behaves like "number", so valueOf() is called.
  • valueOf returns 10 (a primitive value).
    The final operation is string concatenation: '' + 10 = "10".

Difference between toString() and String():

  • Both toString() and String() can be used to convert values into strings.
  • toString()
    • Can convert all data types to strings, except for null/undefined. Calling toString() on null/undefined will throw an error.
    • If the current value is a number, the toString() method can accept a radix parameter, allowing the number to be converted into a string in the specified base.
  • String()
    • Can convert null/undefined into strings.
    • Does not support radix conversion for numbers.

Notes:

  • Both Symbol.toPrimitive and toString must return a primitive value.
  • valueOf may return a primitive value, but it can also return other types.

By default, the preferred type is Number.

  • This means that, in most cases when objects are coerced during operations like addition, JavaScript will attempt to call valueOf first, and then fall back to toString if necessary.

Special case: For the Date object, the default preferred type is String.

JS Design for Object and Array

Object

In JavaScript, the pure Object type has the methods valueOf and toString, and their return values are as follows:

  • valueOf: returns the object itself.
  • toString: returns the string "[object Object]". For different built-in objects, the return value is "[object type]".
    • type refers to the internal type tag of the object. For example, the Math object returns "[object Math]".
    • Some built-in objects override this method, so the return value may differ when called directly. (Note: in the returned string, "object" begins with a lowercase “o,” while the type name begins with an uppercase letter.)

Because of this, the toString method on Object can be used to determine the type of various objects.

Array

Arrays (Array) are very commonly used.

Although they are objects, their design differs from Object, because the toString method is overridden. The return values of the two methods are as follows:

  • valueOf method: returns the array object itself (same as Object).
  • toString method: equivalent to calling join(',') on the array elements and returning the resulting string. For example: [1,2,3].toString(); // "1,2,3"

This behavior is important to keep in mind.

Function

Function objects are rarely used directly, and their toString method is overridden, so it is not the same as Object’s default toString. The return values of the two methods are:

  • valueOf method: returns the function object itself (same as Object).
  • toString method: returns the function’s source code as a string.

Number, String, and Boolean Wrapper Objects

Wrapper objects in JavaScript are special objects designed for the primitive types number, string, and boolean. All the properties and methods used on these three primitive types are actually provided by their corresponding wrapper objects.

The valueOf and toString methods of wrapper objects are overridden on their prototypes, so their return values differ from those of a general Object:

  • valueOf method: returns the corresponding primitive value.
  • toString method: returns the string representation of the primitive value.

The toString methods of these three wrapper objects have specific behaviors:

  • Number wrapper object’s toString method: can take an optional argument specifying the radix/base (e.g., 2, 8, 16) for conversion to string.
  • String wrapper object’s toString method: behaves the same as valueOf, returning the primitive string value.
  • Boolean wrapper object’s toString method: returns the string "true" or "false".

Note: A common source of confusion is the difference between using the wrapper objects themselves (new Number(), new String(), new Boolean()) and using the coercion functions Number(), String(), and Boolean().

  • Wrapper objects must be instantiated with the new keyword, e.g., new Number(123).
  • By contrast, Number('123') is a type conversion function, coercing other types into a number.

In fact, Number(), String(), and Boolean() correspond directly to the internal abstract operations ToNumber, ToString, and ToBoolean defined in the ECMAScript specification.

When converting an object type, these internal operations first apply ToPrimitive (to reduce the object to a primitive value) before performing the final conversion into the target type.

Number + Other Non-String Primitive Types

When a number is added to another type, the operation will use string concatenation only if one operand is a string.

In all other cases, the number takes precedence, and the other primitive type is converted into a number for mathematical addition.

Addition with Primitive Types Other Than Numbers/Strings

When primitive types other than numbers or strings are used directly with the + operator, they are converted to numbers first, and then added. This process has nothing to do with string concatenation.

[] + []

  • When two arrays are added, the conversion still follows the order valueOf -> toString.
  • Since valueOf returns the array object itself, JavaScript then calls toString, which produces the primitive value. For an empty array [], its string conversion result is "".
  • Therefore, the operation is equivalent to adding two empty strings. According to the addition rules (step 2: string concatenation), the result of concatenating two empty strings is another empty string.

{} + {}

  • When two empty objects are added, the conversion still follows the order valueOf -> toString.
  • Since valueOf returns the object itself, JavaScript then calls toString, whose return value is a primitive. For {}, the string conversion result is "[object Object]".
  • Therefore, the operation is equivalent to adding two "[object Object]" strings. According to the addition rules (step 2: string concatenation), the result is string concatenation: "[object Object][object Object]"

Special Note: The expression {} + {} may produce different results in different browsers.

  • In some browsers, such as Firefox and Edge, {} + {} is interpreted as equivalent to +{}. This happens because a leading curly brace { is parsed as the start of a block statement, not as an object literal.
  • As a result, the first {} is ignored, and the entire expression is treated as +{}.
  • This is effectively the same as forcing numeric conversion with Number({}), which is equivalent to Number("[object Object]"), and the result is NaN.

{} + []

The situation described earlier—where {} is interpreted as a block statement — also occurs here, but this time all browsers produce a consistent result.

When {} appears on the left and [] on the right, the left operand is treated as a block statement rather than an object literal.

Thus, {} + [] is equivalent to the expression +[], which forces numeric conversion. This is the same as Number([]), which in turn is equivalent to Number(""), resulting in the number 0.

Note: When the first operand is {}, followed by an array, number, or string, the + operator is interpreted as a unary plus operator, meaning it performs numeric coercion. This is a common pitfall and should be used with caution.

Date Object

The Date object has the following behavior for its valueOf and toString methods:

  • valueOf method: returns the given time as a UNIX timestamp (the number of milliseconds since 1 January 1970 00:00:00 UTC).
  • toString method: returns the localized date/time string.

As mentioned earlier, the Date object is a special case where the preferred type is "string", unlike most other objects (which prefer "number" and therefore call valueOf before toString).

Symbol Type

The Symbol data type was introduced in ES6. It is neither a regular primitive value nor an object, and it does not have built-in implicit type conversion.

Therefore, it cannot be directly used with the + operator. Attempting to do so will result in an error.
